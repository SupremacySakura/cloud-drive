# 约定速记（摘自文档命名与编写约定）

## 1. 命名规则

- 文件名使用小写英文 + 连字符（kebab-case），例如：`project-constraints.md`。
- 文件名应体现“主题”而不是“动作”，避免 `how-to-xxx.md` 这类不稳定命名。
- 索引文件固定为 `docs/README.md`。

## 2. 文档粒度规则

- 一篇文档只覆盖一个一级主题，避免跨主题混写。
- 单篇建议控制在“可一次阅读完成”的长度；超过时先拆子章节，再考虑拆文件。
- 示例、命令、流程仅放在对应主题正文，不回填到索引页。

## 3. 交叉引用规则

- 主题之间通过相对路径链接引用，不复制正文段落。
- 跨文档引用使用“见：`文件名` + 章节名”的形式。
- 若存在唯一权威来源，引用页只放摘要与跳转。

## 4. 单一信息源原则

- 同一规则只在一个正文文档维护。
- 当规则变更时，先更新权威正文，再更新索引页链接或摘要。
- 禁止在多个文件长期维护同一份完整规则。

## 5. 变更同步流程

- Step 1：先定位“权威正文文档”，只在该文档更新完整内容。
- Step 2：同步更新 `docs/README.md` 与相关 `AGENTS.md` 的导航链接或一句话说明。
- Step 3：检查被引用文档路径可达，避免死链与孤立文档。
- Step 4：按场景做一次走读（入门、执行、测试），确认阅读闭环。

## 6. 索引与正文边界补充

- 索引页回答“去哪里看”。
- 正文页回答“具体怎么做”。
