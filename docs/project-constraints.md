# 项目约束

## 来源

- 根目录 `AGENTS.md` 的安全警告与协作约定。
- 后端 `apps/cloud-drive-backend/AGENTS.md` 的安全与配置风险条目。

## 适用范围

- 适用于仓库协作中的硬性规则、风险边界与发布前必须检查项。

## 更新入口

- 当安全配置、环境变量策略、目录路径、团队协作规则变更时，更新本文。
- 若某条约束已有单一权威文档，本文仅保留摘要并给出跳转，不重复维护全文。

## 安全配置约束

- 禁止在代码中长期保留明文敏感信息（数据库账号、JWT 密钥、绝对路径凭据）。
- `internal/utils/jwt.go` 中示例密钥 `your_secret_key` 仅可用于本地开发，不可用于生产环境。
- `internal/database/db.go` 中硬编码 DSN 必须迁移为环境变量配置。
- 对外分享日志、截图、错误信息时，需脱敏 token、用户标识、数据库连接信息。

## 路径与环境约束

- 当前仓库存在硬编码存储路径（`data/` 相关），跨机器运行时优先改为环境变量驱动。
- 前端与后端联调依赖固定端口约定（前端 `5173`、后端默认 `9000`），变更后必须同步文档与配置。
- 包管理统一使用 `pnpm`，避免混用 `npm` 造成锁文件与依赖树不一致。

## 代码与文档同步约束

- 涉及架构、启动命令、目录职责变更时，必须同步更新 `docs/` 对应主题文档。
- 索引页 `docs/README.md` 只维护导航与阅读顺序，不承载长篇正文规则。
- 同一规则只能在一个主题文档作为权威来源维护，其他文档通过链接引用。

## 提交与变更边界

- 变更前明确范围：前端、后端、文档三类改动尽量按主题聚合，减少跨域噪音。
- 未经明确需求，不在一次任务中混入重构式大改（例如目录搬迁、批量命名改动）。
- 提交前至少执行与改动相关的最小验证，参考 `testing-guide.md`。
